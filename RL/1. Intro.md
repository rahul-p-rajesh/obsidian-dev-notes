
## 1.1 What is RL  (**Reinforcement Learning**)
1. It is a loop of
	- observation: what the agent can currently see
	    - Example: game screen, robot camera image, cart position/velocity
	- action: what the agent decides to do
	    - Example: move left, turn right, accelerate
	- reward: a number telling the agent how good that step was
	    - Positive reward usually means “good”
	    - Negative reward usually means “bad”
	- done: whether the episode has ended
	    - Example: game over, robot fell, goal reached
	- policy: the agent’s decision rule
	    - “Given what I see, what should I do?”



## 1.2 Gym loop vs what is OpenEnv service loop
1. From first principles, both loops are doing the same job:
	- there is a world
	- an agent takes actions in that world
	- the world changes
	- the agent learns from the consequences
2. ==The difference is where that “world” lives.==
3. An RL system has 2 main parts:
	- Agent: the thing deciding what to do
	- Environment: the thing the agent interacts with and gets consequence/observations which has rewards associated to it

	
### 1.2.1 What is a GYM Loop then?
1. In the classic Gym style, the environment is just a Python object in the same program.
```python

obs = env.reset() # env is an in-memory Python object
done = False

while not done:
    action = policy.choose(obs)
    next_obs, reward, done, info = env.step(action) #env.step(action) is just a normal function call taking action
    policy.learn(obs, action, reward, next_obs, done) # learns from the result
    obs = next_obs 
    
```
2. Both Agent and env  are living in a single enclosure

#### Disadvantages 
- if the environment crashes, it may kill training too
- tied closely to Python
- harder to package and deploy consistently
- can get messy when scaling across many machines

### 1.2.2 What is a **OpenEnv Service Loop**
1. In a service-based setup, the environment is not just a local Python object. It is a separate service, often in another process or container, accessed through HTTP or RPC.

#### **What is the actual difference?**
Same RL logic, different wiring.

Gym loop:
- environment is a local object
- communication is a function call
- usually Python-only
- very low latency

OpenEnv service loop:
- environment is a separate service
- communication is over network/API
- language-independent
- isolated and easier to deploy
